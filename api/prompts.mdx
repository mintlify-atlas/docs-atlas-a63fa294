## Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/prompts` | No | List prompts with pagination and filters |
| POST | `/api/prompts` | Yes | Create a new prompt |
| GET | `/api/prompts/:id` | No | Get a single prompt by ID |
| PATCH | `/api/prompts/:id` | Yes | Update a prompt (must be author or admin) |
| DELETE | `/api/prompts/:id` | Yes | Delete a prompt (must be author or admin) |

---

## GET /api/prompts

List public, non-unlisted prompts. Results are paginated.

### Query parameters

<ParamField query="page" type="number" default="1">
  Page number (1-indexed).
</ParamField>

<ParamField query="perPage" type="number" default="24">
  Number of results per page.
</ParamField>

<ParamField query="type" type="string">
  Filter by prompt type. One of `TEXT`, `IMAGE`, `VIDEO`, `AUDIO`, `SKILL`, `TASTE`.
</ParamField>

<ParamField query="category" type="string">
  Filter by category ID.
</ParamField>

<ParamField query="tag" type="string">
  Filter by tag slug. Comma-separate multiple slugs to require all of them (AND logic), e.g. `coding,productivity`.
</ParamField>

<ParamField query="sort" type="string" default="newest">
  Sort order. One of `newest`, `oldest`, or `upvotes`.
</ParamField>

<ParamField query="q" type="string">
  Keyword search across title, content, and description.
</ParamField>

### Response

```json
{
  "prompts": [...],
  "total": 1240,
  "page": 1,
  "perPage": 24,
  "totalPages": 52
}
```

  <ResponseField name="title" type="string">Prompt title.</ResponseField>

  <ResponseField name="description" type="string">Short description (up to 500 characters).</ResponseField>
  <ResponseField name="content" type="string">The full prompt text.</ResponseField>
  <ResponseField name="type" type="string">Output type: `TEXT`, `IMAGE`, `VIDEO`, `AUDIO`, `SKILL`, or `TASTE`.</ResponseField>
  <ResponseField name="structuredFormat" type="string">Input format indicator: `JSON` or `YAML`. Null if not structured.</ResponseField>
  <ResponseField name="mediaUrl" type="string">URL of an associated media file.</ResponseField>
  <ResponseField name="requiresMediaUpload" type="boolean">Whether this prompt requires a media upload.</ResponseField>
  <ResponseField name="requiredMediaType" type="string">Type of media required: `IMAGE`, `VIDEO`, or `DOCUMENT`.</ResponseField>
  <ResponseField name="requiredMediaCount" type="number">Number of media items required (1â€“10).</ResponseField>
  <ResponseField name="bestWithModels" type="string[]">Up to 3 model slugs this prompt works best with.</ResponseField>
  <ResponseField name="bestWithMCP" type="object[]">MCP server configs recommended for this prompt. Each item has `command` (string) and optional `tools` (string[]).</ResponseField>
  <ResponseField name="workflowLink" type="string">URL to test or demo the workflow.</ResponseField>
  <ResponseField name="viewCount" type="number">Number of times the prompt has been viewed.</ResponseField>
  <ResponseField name="voteCount" type="number">Total number of upvotes.</ResponseField>
  <ResponseField name="contributorCount" type="number">Number of contributors.</ResponseField>

  <ResponseField name="author" type="object">
    <Expandable title="author fields">
      <ResponseField name="id" type="string">Author user ID.</ResponseField>

      <ResponseField name="username" type="string">Username.</ResponseField>
      <ResponseField name="avatar" type="string">Avatar URL.</ResponseField>
      <ResponseField name="verified" type="boolean">Whether the author is verified.</ResponseField>

  </ResponseField>
  <ResponseField name="category" type="object">Category object with `id`, `name`, `slug`, `icon`, and optional `parent`.</ResponseField>
  <ResponseField name="tags" type="object[]">Array of prompt-tag join objects. Each has a `tag` property with `id`, `name`, `slug`, and `color`.</ResponseField>

## POST /api/prompts

Create a new prompt. Requires authentication.

### Request body

<ParamField body="title" type="string" required>
  Prompt title (1â€“200 characters).
</ParamField>

<ParamField body="content" type="string" required>
  The full prompt text.
</ParamField>

<ParamField body="type" type="string" required>
  Output type: `TEXT`, `IMAGE`, `VIDEO`, `AUDIO`, `SKILL`, or `TASTE`.
</ParamField>

<ParamField body="tagIds" type="string[]" required>
  Array of tag IDs to attach. Pass an empty array if no tags.
</ParamField>

<ParamField body="isPrivate" type="boolean" required>
  Set to `true` to make the prompt visible only to you.
</ParamField>

<ParamField body="description" type="string">
  Short description (up to 500 characters).
</ParamField>

<ParamField body="categoryId" type="string">
  ID of the category to assign.
</ParamField>

<ParamField body="structuredFormat" type="string">
  Input format indicator: `JSON` or `YAML`.
</ParamField>

<ParamField body="contributorIds" type="string[]">
  Array of user IDs to credit as contributors.
</ParamField>

<ParamField body="mediaUrl" type="string">
  URL of an associated media file.
</ParamField>

<ParamField body="requiresMediaUpload" type="boolean">
  Whether users need to upload media to use this prompt.
</ParamField>

<ParamField body="requiredMediaType" type="string">
  Required media type: `IMAGE`, `VIDEO`, or `DOCUMENT`.
</ParamField>

<ParamField body="requiredMediaCount" type="number">
  Number of media items required (1â€“10).
</ParamField>

<ParamField body="bestWithModels" type="string[]">
  Up to 3 model slugs this prompt works best with.
</ParamField>

<ParamField body="bestWithMCP" type="object[]">
  MCP server configs recommended for this prompt. Each object requires `command` (string) and accepts optional `tools` (string[]).
</ParamField>

<ParamField body="workflowLink" type="string">
  URL to test or demo the workflow.
</ParamField>

### Error responses

| Status | Error | Cause |
|--------|-------|-------|
| 401 | `unauthorized` | Not authenticated |
| 400 | `validation_error` | Request body fails schema validation |
| 409 | `duplicate_prompt` | You already have a prompt with the same title or content |
| 409 | `content_exists` | A different author already has a prompt with similar content |
| 429 | `rate_limit` | Last prompt was created fewer than 30 seconds ago |
| 429 | `daily_limit` | Flagged account has reached 5 prompts today |

## GET /api/prompts/:id

Retrieve a single prompt by its ID. Private prompts are only returned for their author.

### Path parameters

<ParamField path="id" type="string" required>
  The prompt ID.
</ParamField>

### Response

Returns the full prompt object (same fields as the list endpoint) plus:

<ResponseField name="hasVoted" type="boolean">Whether the authenticated user has voted on this prompt. `false` if not authenticated.</ResponseField>
<ResponseField name="versions" type="object[]">Up to 10 most recent version history entries.</ResponseField>

### Error responses

| Status | Error | Cause |
|--------|-------|-------|
| 404 | `not_found` | Prompt does not exist or is deleted |
| 403 | `forbidden` | Prompt is private and you are not the author |

## PATCH /api/prompts/:id

Update an existing prompt. You must be the author or an admin.

All fields are optional â€” include only the fields you want to change.

### Path parameters

<ParamField path="id" type="string" required>
  The prompt ID.
</ParamField>

### Request body

Same fields as POST, all optional. Omit any field you don't want to change.

### Error responses

| Status | Error | Cause |
|--------|-------|-------|
| 401 | `unauthorized` | Not authenticated |
| 403 | `forbidden` | Not the author or admin |
| 404 | `not_found` | Prompt does not exist |
| 400 | `validation_error` | Request body fails validation |

## DELETE /api/prompts/:id

Soft-delete a prompt. The prompt is marked with a `deletedAt` timestamp and excluded from all public responses.

<Note>
  Prompts are released under CC0 (public domain). Owners can only delete their own prompts if they have been delisted for quality issues. Admins can delete any prompt. Contact an admin for other deletion requests.
</Note>

### Path parameters

<ParamField path="id" type="string" required>
  The prompt ID.
</ParamField>

### Response

```json
{
  "success": true,
  "message": "Delisted prompt deleted successfully"
}
```

### Error responses

| Status | Error | Cause |
|--------|-------|-------|
| 401 | `unauthorized` | Not authenticated |
| 403 | `forbidden` | Not permitted to delete this prompt |
| 404 | `not_found` | Prompt does not exist |
| 400 | `already_deleted` | Prompt is already deleted |
