## Option 1: Use the web app

The fastest way to get started is at [prompts.chat](https://prompts.chat) — no installation needed.

<Steps>
  <Step title="Browse prompts">
    Visit [prompts.chat/prompts](https://prompts.chat/prompts) to explore the community library. Filter by category, tag, or AI model to find what you need.
  </Step>
  <Step title="Copy a prompt">
    Click the copy icon on any prompt card to instantly copy it to your clipboard. Paste it into ChatGPT, Claude, Gemini, or any AI assistant.
  </Step>
  <Step title="Sign in to save favorites">
    Click **Sign in** and authenticate with GitHub or Google. Once signed in, you can upvote prompts, save them to collections, and create your own.
  </Step>
  <Step title="Create your first prompt">
    Go to [prompts.chat/prompts/new](https://prompts.chat/prompts/new) and fill in the title, description, and prompt content. Choose a category and tags, then click **Create**.
  </Step>
</Steps>

## Option 2: Use the MCP server

Connect prompts.chat to your AI tools via the [Model Context Protocol](https://modelcontextprotocol.io).

<Tabs>
  <Tab title="Remote (recommended)">
    Add the following to your MCP client configuration (e.g., Claude Desktop, Cursor):

    ```json
    {
      "mcpServers": {
        "prompts.chat": {
          "url": "https://prompts.chat/api/mcp"
        }
      }
    }
    ```

    This connects directly to the hosted prompts.chat MCP server — no local installation required.
  </Tab>
  <Tab title="Local (npx)">
    Run the MCP server locally using npx:

    ```json
    {
      "mcpServers": {
        "prompts.chat": {
          "command": "npx",
          "args": ["-y", "prompts.chat", "mcp"]
        }
      }
    }
    ```
  </Tab>
</Tabs>

Once connected, your AI tool can search and retrieve prompts directly from prompts.chat using tools like `search_prompts` and `get_prompt`.

## Option 3: CLI

Run prompts.chat from the terminal with no installation:

```bash
npx prompts.chat
```

This opens an interactive CLI to browse and copy prompts.

## Option 4: Self-host your own instance

Deploy a private prompt library for your team or organization:

<Steps>
  <Step title="Create a new instance">
    ```bash
    npx prompts.chat new my-prompt-library
    cd my-prompt-library
    ```
    This clones the repository and launches the interactive setup wizard.
  </Step>
  <Step title="Configure branding and auth">
    The setup wizard guides you through:
    - App name, logo, and description
    - Authentication provider (GitHub, Google, or email/password)
    - Languages and feature flags
  </Step>
  <Step title="Set environment variables">
    ```bash
    cp .env.example .env
    # Edit .env with your database URL and auth credentials
    ```
  </Step>
  <Step title="Run the database setup">
    ```bash
    npm run db:migrate
    npm run db:seed
    ```
  </Step>
  <Step title="Start the server">
    ```bash
    # Development
    npm run dev

    # Production
    npm run build && npm run start
    ```
  </Step>
</Steps>

<Info>
  For Docker deployment, see the [Docker guide](/self-hosting/docker). For full configuration options, see [Self-Hosting Overview](/self-hosting/overview).
</Info>
